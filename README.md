Android-застосунок для керування футбольними турнірами. Дозволяє створювати команди, формувати стартові склади на інтерактивному полі, проводити матчі в режимі реального часу (лайв-чат подій) та автоматично генерувати турнірну таблицю і статистику гравців (MVP, бомбардири).

## 🛠 Технології
* **Мова:** Kotlin
* **UI:** XML Layouts (Material Design elements, Custom Adapters)
* **База даних:** Firebase Realtime Database (хмарна NoSQL БД)
* **Архітектура:** MVC (Multi-Activity з використанням ActivityResultContracts)

## 🚀 Запуск проєкту на локальному комп'ютері

Проєкт використовує хмарну базу даних Firebase. З міркувань безпеки, файл конфігурації з API-ключами (`google-services.json`) **видалено з репозиторію**. 
Без цього файлу проєкт не збереться в Android Studio і не зможе підключитися до бази.

Щоб запустити проєкт, вам потрібно підключити власну базу Firebase:

### Крок 1: Створення бази даних
1. Перейдіть у [Firebase Console](https://console.firebase.google.com/) та створіть новий проєкт.
2. Додайте Android-додаток до проєкту. Package name має збігатися з тим, що вказаний у `app/build.gradle.kts` (наприклад, `com.example.tournament`).
3. Завантажте згенерований файл **`google-services.json`**.
4. Помістіть цей файл у папку `app/` у корені вашого локального проєкту.

### Крок 2: Налаштування Realtime Database
1. У консолі Firebase відкрийте розділ **Realtime Database** і натисніть "Create Database".
2. Оберіть локацію (наприклад, Europe) і почніть у **Test Mode**.
3. Перейдіть на вкладку **Rules** і встановіть правила доступу. Для повноцінного тестування (читання та запис) правила мають виглядати так:
```bash
{
  "rules": {
    ".read": "true",
    ".write": "true"
  }
}
```

Примітка: Firebase Test Mode автоматично блокує базу через 30 днів. Якщо додаток перестав зберігати дані, перевірте, чи не закінчився термін дії правил.

Крок 3: Збірка в Android Studio
Відкрийте проєкт в Android Studio.

Дочекайтеся завершення синхронізації Gradle (Sync Project with Gradle Files).

Запустіть проєкт на емуляторі або фізичному пристрої, натиснувши Run (Shift+F10).

📂 Структура даних (Firebase)
Додаток не потребує SQL-скриптів для ініціалізації. Firebase створить необхідні гілки автоматично при першому збереженні даних.
Структура бази включає три основні вузли:

teams — інформація про команди та їхні очки.

players — профілі гравців, прив'язка до команд та їхня статистика.

matches — історія ігор, результати та текстовий лог подій матчу.
